# Постер - Медиахаб

Платформа для совместного планирования, создания и публикации контента в социальных сетях. Командная работа над постами, визуальный холст для брейншторма, кросс-постинг в VK и Telegram, аналитика и AI-ассистент.

## Возможности

### Управление контентом
- **Пространства (workspaces)** — изолированные рабочие области с собственными участниками, ролями и социальными аккаунтами.
- **Посты и черновики** — создание, редактирование, планирование и автоматическая публикация по расписанию.
- **Группы постов и варианты** — один пост можно адаптировать под разные платформы.
- **Категории и теги** — структурирование контента по рубрикам с цветовой разметкой.
- **Шаблоны** — переиспользуемые заготовки для типовых публикаций (анонсы, итоги, вакансии, гранты).
- **Календарь публикаций** — наглядный график выходящего контента.

### Совместная работа
- **Холст (Canvas)** — бесконечный realtime-whiteboard для команды: фигуры, текст, стикеры, рисование, изображения, медиа-карточки (YouTube/VK/Telegram), карточки постов.
- **Live-курсоры и presence** — видно, кто из участников сейчас работает на холсте.
- **Задачи** — назначение работ копирайтерам и дизайнерам с дедлайнами и напоминаниями.
- **Предложенные посты** — участники с ролью viewer могут предлагать идеи на ревью.
- **Комментарии к постам** — обсуждение прямо в карточке.
- **Приглашения по ссылке** — гибкая система ролей: owner / admin / editor / viewer.

### Социальные сети
- **VK и Telegram** — подключение аккаунтов, безопасное хранение токенов, автопубликация.
- **Планировщик** — фоновый запуск отложенных публикаций с retry при ошибках.
- **Аналитика** — сбор просмотров и реакций, графики по постам, экспорт в PDF.

### AI-ассистент
- Генерация и переработка текстов постов.
- Подсказки и идеи для контент-плана.
- Проверка орфографии и стиля.

### Уведомления
Внутренние уведомления о запланированных публикациях, выходе постов, ошибках, новых задачах и предложениях.

## Технологический стек

- **Frontend**: React, TypeScript, TanStack Start v1 (SSR), TanStack Router, TanStack Query
- **Сборка**: Vite
- **UI**: Tailwind CSS v4
- **Backend**: Supabase (PostgreSQL, Auth, Storage, Realtime, Edge Functions)
- **Деплой**: Cloudflare Workers (edge runtime)
- **AI**: интеграция с LLM-провайдерами через серверные функции

## Запуск

### Требования
- [Bun](https://bun.sh/) (рекомендуется) или Node.js 20+
- Подключённый Supabase-проект

### Установка

```bash
bun install
```

### Переменные окружения

Файл `.env` создаётся автоматически и содержит:

```
VITE_SUPABASE_URL=...
VITE_SUPABASE_PUBLISHABLE_KEY=...
VITE_SUPABASE_PROJECT_ID=...
```

### Разработка

```bash
bun run dev
```

Приложение поднимется на `http://localhost:5173`.

### Сборка

```bash
bun run build
```

## Модель данных

Ключевые сущности (см. `supabase/migrations/`):

- `workspaces`, `workspace_users`, `workspace_invites` — пространства и доступы
- `posts`, `post_groups`, `post_variants` — контент и его варианты под платформы
- `categories`, `tags` — рубрикация
- `post_tasks`, `post_task_watchers` — задачи и наблюдатели
- `suggested_posts` — предложения от viewer-ов
- `social_accounts`, `workspace_social_accounts` — подключённые соцсети
- `post_analytics` — метрики публикаций
- `canvas_elements` — элементы realtime-холста
- `notifications` — уведомления пользователей
- `templates` — шаблоны постов
- `creative_sessions`, `creative_messages`, `creative_sources` — AI-сессии
- `profiles`, `user_roles` — пользователи и роли

Безопасность данных обеспечивается Row Level Security и SECURITY DEFINER функциями (`is_workspace_member`, `can_edit_workspace`, `can_admin_workspace`, `has_role`).

## Роли

### Глобальные (`app_role`)
- `admin` — администрирование платформы
- `editor`, `viewer`

### Внутри пространства (`workspace_role`)
- `owner` — полный контроль, удаление пространства
- `admin` — управление участниками, настройками, контентом
- `editor` — создание и публикация постов
- `viewer` — чтение и предложение постов на ревью

## Публикация в соцсети

Планировщик дёргается через публичный endpoint `api/public/scheduler/run`, защищённый секретом. Подходит для вызова из pg_cron или внешнего планировщика. Токены соцсетей хранятся в зашифрованном виде (`encrypted_token`) с использованием серверного ключа.
